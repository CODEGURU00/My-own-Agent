Tool	Description

🤖 Antigravity Agent Crew Configuration (crew_config.yaml)
Define the crew that will execute all tasks. Use CrewAI with a sequential process, but with the ability to loop on errors.

yaml
agents:
  - name: Architect
    role: Senior Software Architect
    goal: Break down user requirements into a detailed, implementable plan
    backstory: >-
      You are a master architect who reads the knowledge base and produces
      a technical specification with a file tree and technology choices.
    llm: ollama/devstral-small-2
    tools: [search_knowledge]
    
  - name: FullstackDeveloper
    role: Fullstack Engineer
    goal: Write all code following the Architect’s plan, file by file
    backstory: >-
      You are an expert in web, mobile, and AI integration. You use the
      Nvidia tools when the plan requires it, and write clean, production-ready code.
    llm: ollama/devstral-small-2
    tools: [search_knowledge, create_project, write_file, read_file, bash,
            nvidia_image, nvidia_video, nvidia_voice, nvidia_music]
    
  - name: QAEngineer
    role: Quality Assurance Specialist
    goal: Test the built application, report bugs, and trigger fixes
    backstory: >-
      You run the test suite and the dev server, visually inspect outputs,
      and either approve or send bugs back to the developer.
    llm: ollama/devstral-small-2
    tools: [run_tests, run_dev_server, bash]
    
tasks:
  - name: Understand Requirements
    description: |
      The user’s request: {user_input}.
      Search the knowledge base and output a full architectural plan
      (file structure, tech stack, key dependencies, and steps).
    assigned_agent: Architect
    
  - name: Build Implementation
    description: |
      Follow the Architect’s plan exactly. Generate all code files,
      install dependencies, and create any needed Nvidia media assets.
      Write everything to the project directory.
    assigned_agent: FullstackDeveloper
    context: [Understand Requirements]
    
  - name: Verify and Fix
    description: |
      Run the project’s test suite. If tests fail, report the error
      and return to the Build step with a fix instruction.
      If successful, start the dev server and return the URL.
    assigned_agent: QAEngineer
    context: [Build Implementation]
    
process: sequential
max_iter: 10  # prevent infinite loops
The {user_input} placeholder will be filled by the TUI when you type a command like:
/build a dating app with video chat.
